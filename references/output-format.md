# KKSK Output Format

When applying KKSK, present the work as a progression through the four phases. Use this template for structuring and presenting results.

## Standard Template

```
## Kamae — The Pain
[One-sentence pain statement]

### The Wedge
1. 10x audience: [who is desperate]
2. One dimension: [where you're undeniably better]
3. Saying no to: [what you're refusing]

## Kuzushi — The Workaround
[Current workflow with bottleneck identified]

## Sasoi — The Function
function_name(input_type) → output_type
[Brief description of the transformation]

## Kime — The Build
[The actual code, scoped to the function above]
[Measurement: how will we know this works?]
```

## Guidelines

- Each section should be concise. If a section runs long, it probably contains work that belongs in a different phase.
- The Kamae section is one sentence plus three Wedge Test answers. If it's longer, the pain isn't sharp enough.
- The Kuzushi section is a numbered workflow. If it reads like a paragraph, convert it to steps.
- The Sasoi section is a function signature plus one sentence of explanation. If it needs more, the function isn't isolated.
- The Kime section is code plus a measurement plan. If the measurement plan is vague ("users will like it"), go back to Kamae and find the quantifiable cost.
